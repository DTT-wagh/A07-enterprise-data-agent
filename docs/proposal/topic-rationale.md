# 选题原因与项目价值

> 📌 用于 PPT 答辩前 3 页 + 详细方案文档"项目背景"章节
> 🎯 核心立场：本题精准踩中"AI Agent + 企业数据底座 + 业务自助分析"三大风口，且命题方自身深耕此领域

---

## 一、行业大背景

### 1.1 智能制造的"最后一公里"

- 2025 年国家《"十四五"智能制造发展规划》进入收官年，**90% 规上制造业企业已完成 ERP/MES/QMS 数字化建设**
- 杭州自动化技术研究院服务的客户群体（浙江制造业集群）**已普遍具备数据底座基础**
- 但行业共识：**"数据采集容易，分析困难"** —— 80% 数据处于"沉睡"状态
- 数据从"采集展示"走向"理解决策"成为核心瓶颈

### 1.2 大语言模型的破局点

LLM 在 2023-2025 年发生质变，从"会聊天"到"会用工具"：

| 能力 | 2023 年 | 2025 年 |
|------|---------|---------|
| 自然语言理解 | 通用对话 | **业务领域微调 + 工具调用** |
| 代码生成 | 简单片段 | **多文件工程 + SQL/Python 全栈** |
| 数据分析辅助 | 基础统计 | **多步推理 + 自主规划** |
| 工具使用 | 单工具 | **多工具协同 + ReAct/Plan-Execute** |
| 国产化 | 进口为主 | **DeepSeek / Qwen / ChatGLM 三足鼎立** |

LLM 的成熟为"业务人员自助分析"提供了**第一次可行的技术路径**。

### 1.3 业务侧的迫切需求

> "现场人员不懂 SQL，质量经理想看良率需要排期 3 天等数据工程师响应。"
> —— 某合作制造企业 CIO 在 2024 行业论坛的吐槽

**真实痛点（命题方一线访谈整理）**：
- 生产人员：每班次想看"今天良率 vs 昨日"没有自助手段
- 质量人员：缺陷分析跨多表，SQL 复杂到工程师也要写半天
- 设备人员：停机归因需要关联工艺数据 + 工单 + 报警
- 管理人员：周报/月报靠人工整理，重复劳动巨大

## 二、当前方案的核心痛点（5 个）

| 痛点 | 影响 | 量化数据 |
|------|------|---------|
| 业务人员难以理解数据表/字段/指标 | 自助分析门槛高 | **80%+ 业务人员放弃自助** |
| 用户提出业务问题后难以快速定位数据资源 | 需求响应慢 | 数据团队人均响应周期 **3-5 天** |
| 常见统计/指标计算/图表生成依赖人工 | 重复劳动 | 分析师 **40%** 时间在"取数" |
| 数据分析依赖人工编写 SQL/Python，门槛高 | 人才稀缺 | 制造业数据工程师**缺口 30 万+** |
| 分析结果需人工整理，难以快速形成直观展示 | 决策延迟 | 一份周报从数据到发布 **8 小时+** |

**5 个痛点共同指向一个核心问题**：**"数据丰富但分析能力稀缺"** 的结构性矛盾。

## 三、本项目解决方案

### 3.1 一句话定位

> **面向制造业业务人员的"自然语言问数 + 智能分析"平台，让不懂 SQL 的人也能在 30 秒内完成一次业务分析。**

### 3.2 核心闭环

```
用户提问（自然语言）
   ↓
智能理解（LLM 意图识别 + 业务知识）
   ↓
自动执行（NL2SQL + 数据查询 + 指标计算 + 机器学习）
   ↓
结果展示（表格 + 图表 + 文字结论 + 知识图谱）
   ↓
反哺业务（沉淀问句、指标、口径，持续优化）
```

### 3.3 5 大核心能力（与命题完全对齐）

| 能力 | 命题要求 | 本项目实现 |
|------|---------|-----------|
| ① 业务知识管理 | 业务对象/指标/规则/主题 | **业务知识图谱 + CRUD + 智能检索** |
| ② 数据资源理解 | 表/字段/样例/关系 | **数据资源门户 + 知识图谱可视化** |
| ③ 自然语言智能分析 | NL→SQL→结果 | **LangChain Agent + 8 工具 + 双 LLM** |
| ④ SQL/脚本生成与执行 | 生成+执行+ML | **6 大 ML 算法 + SQL 安全执行** |
| ⑤ 分析结果展示 | 表格+图表 | **ECharts 7 类图表 + 报告生成** |

## 四、为什么是这道题（4 个契合度）

### 4.1 命题方契合

| 维度 | 说明 |
|------|------|
| 命题方背景 | 杭州自动化技术研究院 + 图快数字科技 = **图快"企业大脑 / 数据底座"产品的延伸方向** |
| 业务领域 | 制造业 4 大分析主题（生产/质量/设备/库存）= 命题方深耕的**真实客户场景** |
| 期望能力 | NL2SQL + 知识图谱 + 机器学习 = 命题方**未来 1-2 年路线图** |
| 评分要点 | 业务知识组织 / 数据理解 / NL 分析 / 闭环 / 展示 / 工程 / 应用价值 = **本项目精准命中** |

### 4.2 技术契合

- **LLM 成熟度**：2025 年 ReAct/Plan-Execute 范式稳定，NL2SQL 在 BIRD/SPIDER 基准 SOTA 突破 70%
- **开源生态**：LangChain / LlamaIndex / SQLAlchemy / Vue3 / ECharts **全部成熟国产化或可控**
- **国产化合规**：DeepSeek / Qwen / ChatGLM 完全替代 OpenAI，**符合数据合规要求**
- **比赛窗口**：4 周开发周期内可完成**核心闭环 + 演示兜底**

### 4.3 应用契合

- **目标用户明确**：制造业业务人员（生产/质量/设备/管理）
- **使用场景明确**：日常分析 + 周报/月报 + 专题分析
- **价值可量化**：
  - 自助分析响应时间：**3-5 天 → 30 秒**（提速 **4800 倍**）
  - 数据工程师解放：**40%** 重复性取数工作
  - 业务人员自助率：**20% → 80%**

### 4.4 团队契合

- 3 人学生团队，时间紧（4 周）
- 通过 **MVP 优先 + 双 LLM 兜底 + 录屏兜底** 三层演示保障
- 通过 **角色智能体协作** 最大化单人产出

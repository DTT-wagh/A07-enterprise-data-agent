# 项目特色亮点

> 📌 用于 PPT 答辩"特色亮点"章节 + 详细方案文档"创新点"章节
> 🎯 设计原则：**5 大业务亮点 + 3 大工程亮点 + 2 大演示亮点**，每条亮点都能用一句话讲清 + 一个 demo 场景证明

---

## 一、5 大业务亮点（命题能力精准对齐）

### 亮点 1：业务知识图谱 + 自然语言检索

**一句话**：把"业务对象 / 指标 / 规则 / 主题"织成一张可搜索的知识图谱，让用户用"人话"找知识。

**核心做法**：
- 4 大主题（生产/质量/设备/库存）× 5 大对象（工单/工序/缺陷/停机/物料）× N 个指标（图谱节点）
- 节点带业务标签、口径定义、关联表/字段
- 支持自然语言检索（"良率怎么算"→ 返回定义 + SQL + 关联表）

**Demo 场景**：用户问"不良率是怎么算的" → 系统返回"不良率 = 不良品数 / 检验总数 × 100%"，并附 SQL 模板和关联表 `quality_inspection` + `quality_defect`。

**对应命题要求**：✅ 业务知识管理（业务对象/指标/规则/分析主题）+ ✅ 知识图谱可视化

---

### 亮点 2：多表复杂 NL2SQL + 业务口径自动校准

**一句话**：不是单表 SELECT，而是"工序良率需要 JOIN 三张表"这种业务 SQL 也能自动生成。

**核心做法**：
- LangChain SQL Agent + ReAct 范式 + 多轮反思
- 业务知识注入：把"良率 = 合格数 / 投入数"等口径作为 Agent 的 system prompt
- Schema Linking：自动定位跨表字段（工单 → 工序 → 缺陷）
- 安全护栏：只允许 SELECT / WITH，LIMIT 1000 强制保护

**Demo 场景**：用户问"近 30 天 A 产品的工序不良率" → 自动生成 3 表 JOIN 的 SQL → 跑出结果 → 返回表格 + 柱状图。

**对应命题要求**：✅ 自然语言分析 + ✅ SQL 生成执行 + ✅ 复杂业务逻辑

---

### 亮点 3：业务结果可解释（不只给数据，还给结论）

**一句话**：用户不仅看到一张表，还看到"为什么良率下降"的业务解读。

**核心做法**：
- LLM 对查询结果做"业务解读层"包装：识别趋势 / 异常 / 排名
- 模板化结论：✅ 良率高于均值 5% / ⚠️ 设备 X 停机率异常 / 📈 产线 Y 持续上升
- 关键数字自动高亮（Top3 / Bottom3 / 同比环比）
- 文字结论可一键导出为 Markdown 报告

**Demo 场景**：用户问"本周质量如何" → 系统返回 表格 + 图表 + 文字结论 "本周良率 92.3%，较上周下降 1.2 个百分点，主要原因为 A 工序不良率突增，建议关注。"

**对应命题要求**：✅ 分析结果展示 + ✅ 报告生成 + ✅ 应用价值

---

### 亮点 4：6 大 ML 算法即开即用（业务人员零门槛建模）

**一句话**：业务人员用自然语言就能训练模型、做预测、做异常检测。

**核心做法**：
- 6 大算法（线性回归 / 决策树 / 随机森林 / 逻辑回归 / KMeans / Isolation Forest）封装为 LangChain Tool
- 训练数据自动从数据底座拉取
- 输出：模型文件 + 评估指标 + 特征重要性 + 预测结果
- 异常检测可视化：时间序列 + 异常点标注

**Demo 场景**：用户问"帮我找出近期异常停机的设备" → Isolation Forest 训练 → 输出异常设备 Top5 + 时间分布图。

**对应命题要求**：✅ 建模能力（6 大算法全覆盖）+ ✅ 异常检测 + ✅ 模型解释

---

### 亮点 5：双 LLM 兜底 + 预生成结果 + 录屏兜底（3 层演示保障）

**一句话**：现场演示"绝对不能挂"，我们用 3 层兜底确保万无一失。

**核心做法**：
- **L1 正常**：DeepSeek + Qwen 双供应商，主备自动切换
- **L2 降级**：20 条预生成问句 + 预计算结果缓存，离线也能演示
- **L3 兜底**：5 分钟精简版录屏，电脑故障时启用

**Demo 场景**：现场主问数演示时突然 LLM API 限流 → 系统自动切到 Qwen → 用户无感知；如全部 API 失效 → 一键切换预生成结果；如电脑故障 → 录屏兜底。

**对应命题要求**：✅ 工程可用性 + ✅ 异常处理 + ✅ 答辩演示稳定

## 二、3 大工程亮点（让评委看到"硬功夫"）

### 工程亮点 1：跨平台协作基础设施（Mac/Windows 团队无缝协作）

**一句话**：团队成员用不同操作系统，但代码、配置、CI 完全一致。

**核心做法**：
- 全路径占位符（`${PROJECT_ROOT}` / `${BACKEND_DIR}` 等）替代硬编码
- 跨平台启动脚本（`scripts/dev.sh` + `scripts/dev.ps1` 等价）
- 换行符统一 LF（`.gitattributes`）+ 编辑器规范（`.editorconfig`）
- CI 三平台矩阵（macOS / Windows / Ubuntu）校验换行符、路径硬编码、`.env` 入库

**对应评分**：✅ 工程可用性 + ✅ 应用价值

---

### 工程亮点 2：智能体驱动的开发协作模式

**一句话**：3 个角色智能体（PM/后端/前端/数据/ML）作为"虚拟团队成员"，主控按任务类型自动调度。

**核心做法**：
- 每个智能体有独立 SKILL.md（角色 + 工作流 + 协作契约 + 路径规范）
- 任务类型触发对应智能体（"建表"→ 数据工程师，"建模"→ ML 工程师）
- 智能体输出严格遵循文档结构（PRD / 目录 / 命名）
- 减少 3 人团队的协调成本

**对应评分**：✅ 系统展示 + ✅ 应用价值（智能体方法论本身是创新点）

---

### 工程亮点 3：企业级工程规范

**一句话**：从代码风格到测试覆盖率，从类型检查到安全审计，按"可上线"标准交付。

**核心做法**：
- **代码规范**：Python `ruff` + `mypy`、前端 ESLint + Prettier
- **测试**：pytest 单元 + 集成测试，覆盖率 ≥ 60%
- **安全**：SQL 注入防护（只读 SQL 白名单）、API Key 走环境变量、PII 脱敏
- **可观测性**：结构化日志（structlog）、OpenAPI 文档、错误聚合
- **CI**：三平台矩阵 + secrets 扫描 + 换行符校验

**对应评分**：✅ 工程可用性（最重要评分点之一）

## 三、2 大演示亮点（让评委"哇"出来）

### 演示亮点 1：自然语言"一句话问数"完整闭环

**演示剧本（30 秒）**：
1. 用户在聊天框输入"分析各工序的良率"
2. 系统流式输出"正在理解您的需求... 正在查询数据库..."
3. 自动返回：表格 + 柱状图 + 文字结论
4. 用户继续追问"近 7 天的趋势" → 切换折线图
5. 一键导出 Markdown 周报

**惊艳点**：整个过程 **0 行代码**，业务人员即可操作。

---

### 演示亮点 2：知识图谱可视化（业务全景一目了然）

**演示剧本（30 秒）**：
1. 进入"数据资源"页面
2. 系统渲染知识图谱：4 大主题 → 15 张表 → 50+ 字段 → 关系连线
3. 鼠标 hover 节点显示：表结构 + 样例数据 + 业务口径
4. 点击节点跳转：直接看表的详细字段说明

**惊艳点**：用一张图把"业务对象 ↔ 数据表 ↔ 字段"全部关联，**评委一眼看懂数据底座**。

## 六、答辩话术模板（5 条）

1. **讲亮点 1**："我们没有把业务知识做成静态文档，而是做成可搜索的知识图谱，让系统自己'理解'业务。"

2. **讲亮点 2**："我们不只做单表查询，而是支持跨 3-4 张表的复杂 JOIN，并通过业务口径注入防止 LLM 跑偏。"

3. **讲亮点 3**："我们不只给数据，还给结论 —— 系统会告诉你'良率为什么下降'，这才是真正的业务辅助。"

4. **讲亮点 4**："6 大 ML 算法全部封装为 LangChain Tool，业务人员用'人话'就能建模做预测。"

5. **讲亮点 5**："现场演示不能挂，我们准备了 3 层兜底 —— 双 LLM 切换、预生成结果、录屏兜底，**保证演示零失败**。"
